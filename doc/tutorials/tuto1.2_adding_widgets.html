Introduction

In the first part of this tutorial, you have seen how to create a first basic visualisation of a trace with the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget. In this second part of the tutorial, you will learn how to use other widgets. First, you will add a time scale to the trace. Then you will use a widget that import CSV files in the trace. And finally, you will learn how to use a widget that displays detailed informations about a selected Obsel.

Reminder from part 1 of the tutorial.

Let's start with the code obtained at the end of the previous part of the tutorial: 

<!doctype html>
<html>
	<head>
		<meta charset="utf-8">

Reminders

  • Files "Samotraces.js" and "d3.js" must be placed in a folder called "javascript" and "samotraces.css" in a folder called "css".
  • This code contains a <div> HTML tag element with an attribute id set to "my-widget". This HTML DIV element is where the widget will be instanciated.
  • The Javascript init() function is run when the page is loaded. This function will:
    • Create a trace object, an obsel selector object and a time window object.
    • Add obsels to the trace.
    • Create a widget that visualise the trace.

More about widgets

As you have seen during the first tutorial, when you want to create a {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget, the first argument that will be provided is the ID of the HTML DIV element where the plugin will be instanciated. This will always be the case for all the widgets. Every widget will require a unique ID. The other parameters are specific to each widget, as we will see in this tutorial.

Now, you might wonder what would happen if you try to instanciate two widgets with the same ID. The answer is that it will likely not work. The two widgets might interfere with each other.

In summary, each widget require a unique HTML DIV element with a unique ID. This implies that the first thing to do when adding a widget is to create a new HTML DIV element and create an attribute ID for this element. Then you can update your JavaScript source code to create the new widget.

Adding a time scale

First, we will add a time scale to the trace. This will be done with the {@link Samotraces.UI.Widgets.WindowScale|WindowScale} widget.

The documentation of the {@link Samotraces.UI.Widgets.WindowScale|WindowScale} widget tells us that the {@link Samotraces.UI.Widgets.WindowScale|WindowScale} widget requires two parameters:

  1. A String, which is the ID of the HTML DIV element where we want to instanciate the widget.
  2. A {@link Samotraces.TimeWindow|TimeWindow} which delimitates the current time window.

As we want the time scale to match what is displayed on the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget, we will use the same {@link Samotraces.TimeWindow|TimeWindow} object (tw). All we need to do is to create a HTML DIV element for the widget (with the id "scale" for instance) and then instanciate the {@link Samotraces.UI.Widgets.WindowScale|WindowScale} widget with this id and the {@link Samotraces.TimeWindow|TimeWindow} object tw that have already been instanciated: 

<div id="scale"></div>

new Samotraces.UI.Widgets.WindowScale('scale',tw);

Now you should see something like this:

When zooming in or out on the trace, you should see that the scale updates itself. The same happens if you move the trace to the left of to the right. This is the magic of using the same {@link Samotraces.TimeWindow|TimeWindow} object in the two widgets.

Importing a trace

In the previous tutorial, we had added manually three obsels to an empty trace. In this section, we will import obsels from a CSV file with the {@link Samotraces.UI.Widgets.ImportTrace|ImportTrace} widget.

First thing to do, we have to remove the code that was adding obsels to the trace. Then, we need to instanciate the ImportTrace widget.

Reading the {@link Samotraces.UI.Widgets.ImportTrace|ImportTrace} widget documentation tells us that the {@link Samotraces.UI.Widgets.ImportTrace|ImportTrace} widget requires two parameters:

  1. A String, which is the ID of the HTML DIV element where we want to instanciate the widget.
  2. A {@link Samotraces.Trace|Trace} object in which the CSV file(s) will be imported.

As we want to import the CSV file in the current trace, we only need to create a HTML DIV element for the widget (with the id "importer" for instance) and then instanciate the {@link Samotraces.UI.Widgets.ImportTrace|ImportTrace} widget with this id and the trace object already instanciated in parameter: 

<div id="importer"></div>

new Samotraces.UI.Widgets.ImportTrace('importer',trace);

The resulting code should be the following:

<html>

		<meta charset="utf-8">

		<script type="text/javascript" src="javascript/jquery.js"></script>
		<script type="text/javascript" src="javascript/jquery.mousewheel.js"></script>
		<script type="text/javascript" src="javascript/Samotraces.js"></script>
		<link rel="stylesheet" type="text/css" href="css/samotraces.css"> 
	</head>
	<body>
		<div id="my-widget"></div>
		<div id="scale"></div>

Inspecting an obsel

Now let's visualise the details of an obsel with the {@link Samotraces.UI.Widgets.ObselInspector|ObselInspector} widget. The {@link Samotraces.UI.Widgets.ObselInspector|ObselInspector} widget makes it possible to visualise the content of a selected obsel. It offers a simple list of all the attributes of the selected obsel.

The documentation of the {@link Samotraces.UI.Widgets.ObselInspector|ObselInspector} widget says that it requires two parameters:

  1. A String, which is the ID of the HTML DIV element where we want to instanciate the widget.
  2. A {@link Samotraces.Selector|Selector} of Obsel object.

The obsel Selector object stores the currently selected obsel and informs relevant plugins of the evolution of the selection. Let's try to make the {@link Samotraces.UI.Widgets.ObselInspector|ObselInspector} widget work by adding a new HTML DIV element, and instanciating it with the previously defined {@link Samotraces.Selector|Selector} object: 

<div id="obsel-inspector"></div>

var obsel_selector	= new Samotraces.Selector('Obsel');

new Samotraces.UI.Widgets.ObselInspector('obsel-inspector',obsel_selector);

Now, let's see what happens!

Oh! it seems like nothing new happened... Well, what is this widget supposed to do again? It displays the content of a selected obsel.

Initially, there are no selected obsels. Therefore, there are no obsels to visualise. When there are no obsels to visualise, the default behaviour of the ObselInspector widget is to stay hidden. That is why there are nothing displayed.

You can select an obsel by using the {@link Samotraces.Selector#select|Selector.select()} method, with the obsel as a parameter. However, we would like something more user friendly. For instance, we would like to be able to select an obsel by clicking on it on the visualisation. For this, we need to link the click event on obsels displayed with the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} visualisation widget to the {@link Samotraces.Selector#select|Selector.select()} method.

-- ICI A MODIFIER... -- If you know jQuery, you know it is quite easy to add listen to events and trigger a JavaScript function when an event triggers. Samotraces.js is designed so that is makes it easy for you to use jQuery event functions.

So, how does it work? We will use jQuery to handle the events. With jQuery, you can easily select elements from your page with the jQuery $() function. For instance, you can select the body of the HTML document this way: $('body'). Then, we will add an event listener to the body of the document. This is done with the jQuery .on() method. We will have to write something like this: $('body').on(EVENT_TYPE, SELECTION, CALLBACK), where EVENT_TYPE will be the type of event we are listening to, SELECTION is a jQuery selector that will filter the elements that will be actually listened to, and CALLBACK is a function called when the event is triggered.

Note for jQuery users: When you listen to events with jQuery, you can either listen to events directly triggered on a predefined selection (for instance with $(selection).click()) or add a filter to the selection to trigger the event on that more specific subselection. While the first method is more intuitive and straightforward, it can't be applied in Samotraces, because the elements you want to listen to may not be already existing when you try to create the event listener. Instead, the filtering option allow you to define a behaviour even before everything has been loaded, or even if the content has changed.

Here, the EVENT_TYPE will have to be the "click" event. The SELECTION will be the images corresponding to each obsel. And the CALLBACK is the function that we will call to inspect the obsel. Now, let's define the SELECTION and the CALLBACK.

In Samotraces.js, classes are attached to HTML elements. Inside a Widget, such as the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget, each obsel representation will be attached to the "Σ-obsel" class. Note that "Σ" stands for "samotraces". With jQuery, it is easy to define a class selector (the CSS notation is used: adding a dot before the name of the class). Therefore, our SELECTION argument will be ".Σ-obsel

The CALLBACK function called when an event is triggered requires one argument, which is the event itself. The event contains data, for instance, event.target is the HTML element that has be the target of the event. When clicking on the images that represent an obsel, we would like to get this obsel and to display it. The obsel is not directly accessible from the event. The Obsel object can be obtained with this code: $.data(event.target,'Σ-data') The obsel information is bound to the HTML element that has the "Σ-obsel" class (therefore, the one that is the target of the click event).

If you look more closely to the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget documentation, you will find that the widget triggers an event of type {@link Samotraces.UI.Widgets.TraceDisplayIcons#ui:click:obsel|ui:click:obsel} when the user click on an Obsel. This means, we need to bind the {@link Samotraces.UI.Widgets.TraceDisplayIcons#ui:click:obsel|ui:click:obsel} event to the {@link Samotraces.Selector#select|Selector.select()} method. This can be done with the optional parameters of the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget.

Looking back to the {@link Samotraces.UI.Widgets.TraceDisplayIcons|TraceDisplayIcons} widget documentation, we see that we need to create an object options that contains a field events of type {@link Samotraces.UI.Widgets.EventConfig|EventConfig} in which the behaviour on the {@link Samotraces.UI.Widgets.TraceDisplayIcons#ui:click:obsel|ui:click:obsel} event will be defined. The documentation gives an example of the option paramater, that we can simplify for the purpose of this tutorial: 

var options = {
	events: { "ui:click:obsel": obsel_selector.select.bind(obsel_selector) }
}

// Create widgets
new Samotraces.UI.Widgets.TraceDisplayIcons('my-widget',trace,tw,options);

For those who want to understand more precisely the technical aspect, you might wonder what is the use of the ".bind(obsel_selector)" code. It turns out that with Javascript, if you use the reference to a method this way: obsel_selector.select, the {@link Samotraces.Selector#select|Selector.select()} method will be called, but the this parameter will not be bound to any instance. The .bind(obsel_selector) code ensures that the method will be called on the obsel_selector instance.

<!doctype html>
<html>
	<head>
		<meta charset="utf-8">
		<script type="text/javascript" src="javascript/d3.js"></script>
		<script type="text/javascript" src="javascript/jquery.js"></script>
		<script type="text/javascript" src="javascript/jquery.mousewheel.js"></script>

Going further

As you have seen in this second part of the tutorial, adding widgets is pretty easy. You can easily add functionnalities to your application and add/remove widgets.

You might wonder if the order in which you add the widgets makes a difference on the order in which widgets are displayed. Well, the answer is: the order of the HTML DIV elements makes a difference. The order of the JavaScript code do not make any difference. The reason why is that the positionning of the widgets depends on the HTML flow. If you want to alter the way widget are displayed or the order in which they appear, you will have to edit the HTML code and/or the CSS style-sheets.
